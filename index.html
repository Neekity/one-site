当前判断

    Recommended Stack

    个人测试阶段，不要一上来堆满服务。

    最小可用组合是 Codex + GitHub + Vercel + Supabase + Cloudflare + Resend。Clerk、Pinecone、复杂监控和付费数据库都应该在需求成立后再补。

    7-Day MVP

    验证目录站之前，先验证“决策价值”。

    泛目录很容易被复制。第一周的目标不是收录 500 个工具，而是做出 20 个高质量条目、1 个计算器和 3 套可执行方案。

    1. D1确定唯一细分场景，例如“独立开发者 SaaS 建站成本”。
    2. D2整理 20 个工具条目，字段包含价格、免费额度、商用限制、替代品和坑点。
    3. D3上线计算器，先用静态数据和本地 JS 验证页面是否有收藏价值。
    4. D4补三套方案：个人测试、商业 MVP、增长阶段。
    5. D5接入分析，记录来源、停留、计算器交互和外链点击。
    6. D6在 X、即刻、知乎或小红书发布真实成本拆解。
    7. D7根据收藏率、私信咨询和外链点击决定是否继续做数据库。
